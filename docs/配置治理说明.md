# 配置治理说明

本文档说明项目当前的配置治理方式，覆盖本地开发、Kubernetes 部署和生产环境启动校验。

## 1. 设计原则

- 本地开发允许默认值，降低启动成本。
- 生产环境必须显式提供关键配置，禁止使用默认密码和占位符。
- 敏感配置进入 `Secret`，非敏感配置进入 `ConfigMap`。
- 服务启动早期执行配置校验，缺失或弱配置直接启动失败。
- 连接池、队列、服务地址等运行参数通过环境变量调整，不写死在代码中。

## 2. 环境模式

通过 `APP_ENV` 区分运行环境：

| APP_ENV | 行为 |
|---|---|
| `development` | 默认值可用，适合本地开发 |
| `staging` | 预发布环境，可使用显式配置 |
| `production` | 启用严格配置校验 |

生产严格校验由 `pkg/config.ValidateService` 完成。

## 3. 生产必填配置

### 3.1 公共基础设施

| 配置 | 说明 | Secret/ConfigMap |
|---|---|---|
| `MYSQL_DSN` | MySQL 连接串 | Secret |
| `REDIS_ADDR` | Redis 地址 | ConfigMap |
| `REDIS_PASSWORD` | Redis 密码 | Secret |
| `MQ_URL` | RabbitMQ 连接串 | Secret |
| `JWT_SECRET` | JWT 签名密钥，生产必须至少 32 字符 | Secret |

### 3.2 连接池配置

| 配置 | 默认值 | 说明 |
|---|---:|---|
| `MYSQL_MAX_OPEN_CONNS` | 50 | MySQL 最大打开连接数 |
| `MYSQL_MAX_IDLE_CONNS` | 10 | MySQL 最大空闲连接数 |
| `MYSQL_CONN_MAX_LIFETIME_SECONDS` | 300 | MySQL 连接最大生命周期 |
| `REDIS_POOL_SIZE` | 20 | Redis 连接池大小 |
| `REDIS_MIN_IDLE_CONNS` | 5 | Redis 最小空闲连接数 |
| `REDIS_DB` | 0 | Redis DB |

### 3.3 服务地址

| 配置 | 默认值 |
|---|---|
| `ORDER_GRPC_TARGET` | `localhost:9081` |
| `USER_GRPC_TARGET` | `localhost:9083` |
| `INVENTORY_GRPC_TARGET` | `localhost:9082` |
| `TASK_GRPC_TARGET` | `localhost:9084` |
| `REFUND_GRPC_TARGET` | `localhost:9086` |
| `ACTIVITY_GRPC_TARGET` | `localhost:9087` |
| `PRICE_GRPC_TARGET` | `localhost:9088` |

生产环境中，依赖下游服务的应用必须显式提供对应 target。

## 4. Kubernetes 配置

Kubernetes 配置入口：

- `deploy/k8s/config.yaml`

该文件包含：

- `go-micro-config`：非敏感配置，例如服务地址、连接池参数、MQ 队列名。
- `go-micro-secret`：敏感配置，例如 MySQL DSN、MQ URL、JWT secret。

生产部署前必须替换 `go-micro-secret` 中的占位值。当前代码会拒绝以下生产配置：

- `JWT_SECRET=dev-secret`
- 长度小于 32 的 `JWT_SECRET`
- 包含 `replace-me`、`change-me`、`placeholder` 的占位符
- `MYSQL_DSN` 使用 `root:root`
- `MQ_URL` 使用 `guest:guest`

## 5. 本地开发配置

本地可参考：

- `.env.example`

注意：Go 程序不会自动读取 `.env` 文件。可以用 IDE、shell 或 docker-compose 注入这些变量。

PowerShell 示例：

```powershell
$env:APP_ENV="development"
$env:MYSQL_DSN="root:root@tcp(127.0.0.1:3307)/go_micro?parseTime=true&charset=utf8mb4&loc=Local"
go run ./cmd/gateway-api
```

## 6. 服务校验规则

| 服务 | production 必填 |
|---|---|
| `gateway-api` | `MYSQL_DSN`, `JWT_SECRET`, 全部下游 gRPC target |
| `order-service` | `MYSQL_DSN`, `REDIS_ADDR`, `MQ_URL`, `INVENTORY_GRPC_TARGET` |
| `task-service` | `MYSQL_DSN`, `MQ_URL`, `ORDER_GRPC_TARGET`, `INVENTORY_GRPC_TARGET` |
| `payment-service` | `MYSQL_DSN`, `ORDER_GRPC_TARGET`, `INVENTORY_GRPC_TARGET` |
| `refund-service` | `MYSQL_DSN`, `MQ_URL`, `ORDER_GRPC_TARGET`, `INVENTORY_GRPC_TARGET` |
| `inventory-service` | `MYSQL_DSN`, `REDIS_ADDR` |
| `activity-service` | `MYSQL_DSN`, `REDIS_ADDR` |
| `user-service` | `MYSQL_DSN`, `REDIS_ADDR` |
| `price-service` | `MYSQL_DSN` |

## 7. 后续增强建议

- 为不同环境拆分 `deploy/k8s/overlays/dev|staging|prod`。
- 使用 External Secrets 或云厂商 Secret Manager 管理生产密钥。
- 增加配置变更审计和回滚流程。
- CI 中增加 Kubernetes manifest 校验。
- 将配置校验错误输出为结构化日志，便于平台采集。
