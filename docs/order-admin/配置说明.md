# 订单管理后台 - 系统配置

## 1. 账户信息

### 管理员账户（开发模式）

| 账号 | 密码 | 角色 | 说明 |
|------|------|------|------|
| admin | admin123 | admin | 硬编码开发账户，无需启动 user-service |
| testuser | test123 | user | 需要 user-service 运行，密码存储于 DB（需清除缓存后生效） |

> **说明**: `admin/admin123` 可直接登录使用（网关硬编码）。`testuser/test123` 需要 user-service 通过 gRPC 验证密码。清除 Redis 缓存（`DEL user:uname:testuser`）后即可登录。
>
> 其他用户（如 charlie 等）存在于数据库中，但需要启动 `user-service` 微服务通过 gRPC 调用才能验证登录。

### 登录验证

```bash
# 管理员登录（当前唯一可用）
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

## 6. 主要 API 接口

| 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/v1/auth/login` | POST | 否 | 用户登录 |
| `/api/v1/admin/orders` | GET | Bearer Token | 订单列表 |
| `/api/v1/order-views/:order_no` | GET | Bearer Token | 订单详情 |
| `/api/v1/admin/dashboard/stats` | GET | Bearer Token | 运营看板统计 |
| `/api/v1/payments/:id` | GET | Bearer Token | 支付信息 |

### 认证示例

```bash
# 登录获取 Token
TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' | jq -r '.data.token')

# 使用 Token 访问受保护接口
curl -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/v1/admin/orders?page=1&pageSize=10
```

## 8. 常见问题

### Q: 登录提示"服务不可用"
A: 检查 Gateway 是否运行：`curl http://localhost:8080/api/v1/auth/login`

### Q: testuser 登录提示"invalid username or password"
A: 用户密码缓存在 Redis 中，清除缓存后重试：
```bash
docker exec go-micro-redis-1 redis-cli -n 0 DEL "user:uname:testuser"
```

### Q: 前端页面空白或报错
A: 检查：
1. Gateway 是否运行在 8080 端口
2. 浏览器控制台是否有错误
3. 尝试刷新页面

### Q: 订单列表为空
A: 确保 order-service 和 gateway 都在运行，且数据库中有订单数据。

### Q: 如何查看 API 文档？
A: 访问 http://localhost:8080/swagger/ （需启用 swagger 服务）
