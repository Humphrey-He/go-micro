# 项目说明文档

本文档基于当前代码实现与规划内容整理，作为交付说明与面试口径补充。

## 一、项目概述
- 业务场景：电商下单履约
- 架构模式：Go 微服务 + HTTP（对外）+ gRPC（内部）+ MQ（异步）
- 目标：完整演示订单创建主链路、库存预占、异步履约任务编排及补偿思路

## 二、服务清单与职责
- gateway-api：统一入口、鉴权、路由、Swagger 对外文档
- order-service：订单创建与状态更新、Outbox 事件
- inventory-service：库存预占/释放/确认、库存查询
- user-service：用户信息维护与查询
- task-service：消费订单事件，创建履约任务，重试与死信补偿

## 三、核心链路
1. 用户通过 gateway-api 创建订单
2. order-service 写订单 + 明细
3. order-service 调用 inventory-service 预占库存（gRPC）
4. order-service 事务写入 outbox（order_reserved）
5. outbox 定时发布到 MQ
6. task-service 消费消息生成履约任务（FULFILL）
7. task-service 在履约成功后创建 15 分钟超时取消任务（TIMEOUT_CANCEL）
8. 超时任务触发后若订单未完成，调用取消并释放库存

## 四、关键技术点
### 1. 内部调用（gRPC）
- 统一 IDL，强约束接口
- 更低延迟、连接复用
- 易于治理（超时、重试、拦截器）

### 2. 消息模型（RabbitMQ）
- 工作队列模型，手动 ACK
- 失败消息进入 DLQ
- 主交换机 `order.events`，死信交换机 `order.events.dlx`

### 3. Outbox 一致性
- 订单状态更新与 outbox 插入同一事务
- 由后台轮询发布，保证最终一致性

### 4. 缓存稳定性
- 空值缓存防穿透
- TTL 抖动防雪崩
- 热点变更主动删除

## 五、接口入口
- gateway-api：`/api/v1/orders`
- gateway-api：`/api/v1/order-views/{order_no}`
- user-service：`/users`
- inventory-service：`/inventory/*`
- task-service：`/tasks`

## 六、状态与补偿
- 订单状态机：`CREATED -> RESERVED -> PROCESSING -> SUCCESS`
- 取消规则：仅允许 `CREATED` / `RESERVED` 取消（已取消幂等返回成功）
- 超时取消：15 分钟超时任务触发，若订单未完成则取消并释放库存
- 库存释放：支持 `reserved_id` 或 `order_id` 幂等释放

## 六、运行依赖
- MySQL 8
- Redis
- RabbitMQ

## 七、配置项（核心）
- `MYSQL_DSN`：MySQL 连接串
- `REDIS_ADDR`：Redis 地址
- `MQ_URL`：RabbitMQ 连接串
- `ORDER_GRPC_ADDR` / `INVENTORY_GRPC_ADDR`

## 八、Swagger 文档
- 生成命令：
  ```bash
  swag init -g cmd/gateway-api/main.go -o ./docs/swagger
  ```
- 访问地址：`http://localhost:8080/swagger/index.html`

## 十、测试
CI 已配置：`.github/workflows/ci.yml`  
执行项：
- `go test ./... -v -cover`
- `go test ./... -race`

单元测试覆盖：状态映射、库存幂等释放、timeout cancel 链路  
建议本地执行：
```bash
go test ./... -v -cover -race
```
说明：
- `-race` 需要启用 CGO 且系统具备 C 编译器（Windows 需安装 gcc）
- 如果启用 Go Toolchain 下载，请确保网络可达或设置 `GOTOOLCHAIN=local`
