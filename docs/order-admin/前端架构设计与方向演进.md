# 前端架构设计与方向演进

> 本文档为订单管理后台前端的技术架构设计，涵盖技术选型、架构模式、目录结构及演进路径。

---

## 1. 技术选型

### 1.1 核心框架

| 层级 | 选项 A（推荐） | 选项 B | 选项 C |
|------|----------------|--------|--------|
| 框架 | **React 18 + Vite** | Vue 3 + Vite | Next.js 14 |
| UI 组件库 | **Ant Design 5.x** | Element Plus | shadcn/ui |
| 状态管理 | **Zustand** | Pinia | Redux Toolkit |
| 路由 | **React Router 6** | Vue Router | Next.js Router |
| HTTP 客户端 | **Axios** | Fetch API | tRPC |

**推荐理由：**

- **React + Vite**：生态成熟，开发体验好，HMR 快
- **Ant Design 5**：企业级组件丰富，支持 Design Token，主题定制能力强
- **Zustand**：轻量级状态管理，比 Redux 少很多模板代码
- **Axios**：拦截器、请求取消、自动重试等特性完善

### 1.2 辅助库

| 用途 | 库 |
|------|---|
| 图表 | ECharts / Recharts |
| 日期处理 | Day.js |
| 表单验证 | React Hook Form + Zod |
| 国际化 | i18next |
| 工具库 | Lodash-es |
| CSS 方案 | Tailwind CSS 或 CSS Modules |

## 3. 核心模块设计

### 3.1 API 层设计

```typescript
// src/api/request.ts
import axios from 'axios';

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000,
});

// 请求拦截器：注入 JWT
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('admin_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器：统一错误处理
request.interceptors.response.use(
  (response) => response.data,
  (error) => {
    // 统一错误处理：401 跳转登录、500 提示等
    return Promise.reject(error);
  }
);

export default request;
```

```typescript
// src/features/orders/ordersApi.ts
import request from '@/api/request';
import type { OrderListParams, OrderListResponse } from './types';

export const getOrderList = (params: OrderListParams) =>
  request.get<OrderListResponse>('/api/v1/admin/orders', { params });

export const getOrderDetail = (orderNo: string) =>
  request.get(`/api/v1/order-views/${orderNo}`);

export const cancelOrder = (orderId: string) =>
  request.post(`/api/v1/orders/${orderId}/cancel`);
```

### 3.2 状态管理设计

```typescript
// src/stores/authStore.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface AuthState {
  token: string | null;
  userInfo: { userId: string; username: string; role: string } | null;
  setAuth: (token: string, userInfo: AuthState['userInfo']) => void;
  logout: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      token: null,
      userInfo: null,
      setAuth: (token, userInfo) => set({ token, userInfo }),
      logout: () => set({ token: null, userInfo: null }),
    }),
    { name: 'admin-auth' }
  )
);
```

### 3.3 通用列表模式

```typescript
// 通用列表 Hook
export const useOrderList = () => {
  const [loading, setLoading] = useState(false);
  const [data, setData] = useState<Order[]>([]);
  const [pagination, setPagination] = useState({ page: 1, pageSize: 20, total: 0 });

  const fetchData = useCallback(async (params: OrderListParams) => {
    setLoading(true);
    try {
      const res = await getOrderList(params);
      setData(res.data);
      setPagination(res.pagination);
    } finally {
      setLoading(false);
    }
  }, []);

  return { loading, data, pagination, fetchData, setPagination };
};
```

### 3.4 权限控制

```typescript
// 路由守卫
const ProtectedRoute = ({ children, requiredRoles }: { children: ReactNode; requiredRoles?: string[] }) => {
  const { token, userInfo } = useAuthStore();

  if (!token) {
    return <Navigate to="/login" replace />;
  }

  if (requiredRoles && !requiredRoles.includes(userInfo?.role || '')) {
    return <Navigate to="/403" replace />;
  }

  return children;
};

// 按钮级权限
const PermissionButton = ({ action, children }: { action: string; children: ReactNode }) => {
  const { userInfo } = useAuthStore();
  const hasPermission = checkPermission(userInfo?.role, action);

  if (!hasPermission) return null;
  return <>{children}</>;
};
```

## 4. 方向演进

### Phase 1：单体前端（MVP）

**目标**：快速交付可用的管理后台

```
技术栈：React + Ant Design + Zustand + Axios
部署：Nginx 单实例
```

**特点**：
- 所有功能在一个前端项目中
- 前端直接调用 Gateway API
- 简单够用，快速验证业务

---

### Phase 2：微前端拆分

**触发条件**：
- 团队扩大，多人同时开发
- 某些模块需要独立发布
- 不同模块技术栈差异大

**演进方案**：采用 **Module Federation**（Webpack 5）或 **Qiankun**

```
┌──────────────────────────────────────────┐
│              Container App               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐    │
│  │  Order  │ │ Payment │ │Inventory│    │
│  │  App    │ │  App    │ │  App    │    │
│  └─────────┘ └─────────┘ └─────────┘    │
└──────────────────────────────────────────┘
```

**目录演进**：

```
独立拆分后：
├── order-admin/              # 主应用容器
├── order-admin-orders/       # 订单子应用
├── order-admin-payments/      # 支付子应用
└── order-admin-inventory/    # 库存子应用
```

---

### Phase 3：BFF 层

**触发条件**：
- 前端需要聚合多个微服务数据
- 减少前端与后端的接口耦合
- 需要统一做数据转换/缓存

**演进方案**：新增 **BFF（Backend For Frontend）** 服务

```
┌─────────────┐
│   Browser   │
└──────┬──────┘
       │ GraphQL / REST
       ▼
┌─────────────────────────────────────────┐
│           BFF Service (Node.js)          │
│   ┌─────────────────────────────────┐   │
│   │  GraphQL Schema / REST Aggr.   │   │
│   └─────────────────────────────────┘   │
└──────┬──────────────────────┬──────────┘
       │ gRPC                  │ gRPC
       ▼                       ▼
┌─────────────┐         ┌─────────────┐
│Order Service│         │Payment Svc  │
└─────────────┘         └─────────────┘
```

**备选方案**：
- Node.js + Express/Koa + Apollo Server
- NestJS + GraphQL
- 或者直接在 Gateway 层扩展聚合接口（Go）

## 5. 技术债务与风险

| 风险 | 应对 |
|------|------|
| 登录 Token 与用户端共用 | 独立管理员 JWT 体系，区分 `admin` 角色 |
| 接口无权限控制 | 后端新增 Admin API Middleware |
| 数据量大无分页 | 前端强制分页，后端加 Limit |
| 移动端适配 | MVP 仅支持 PC，Phase 2 后考虑响应式 |

## 7. 总结

| 阶段 | 复杂度 | 适用场景 | 交付速度 |
|------|--------|----------|----------|
| Phase 1 单体 | 低 | 小团队 MVP | 快 |
| Phase 2 微前端 | 中 | 多团队并行 | 中 |
| Phase 3 BFF | 中高 | 前后端分离协作 | 中 |
| Phase 4 工程化 | 低 | 长期维护 | - |

**建议**：从 Phase 1 单体起步，使用 React + Ant Design + Zustand，保持目录结构按功能模块组织（features/），为后续拆分到微前端做好准备。
