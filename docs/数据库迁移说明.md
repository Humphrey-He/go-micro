# 数据库迁移说明

本文档说明项目的数据库迁移管理方式。

## 1. 迁移目录

版本化迁移文件位于：

```text
deploy/migrations/
```

当前初始版本：

```text
000001_init_schema.up.sql
000001_init_schema.down.sql
```

`deploy/sql/schema.sql` 暂时保留，用于本地快速初始化和兼容旧文档；生产和 CI 应优先使用 `deploy/migrations`。

## 2. 工具约定

默认使用 `golang-migrate` CLI。

安装示例：

```bash
go install -tags "mysql" github.com/golang-migrate/migrate/v4/cmd/migrate@latest
```

## 3. Makefile 命令

执行迁移：

```bash
make migrate-up MIGRATE_DATABASE='mysql://user:password@tcp(localhost:3306)/go_micro?multiStatements=true&parseTime=true'
```

回滚一个版本：

```bash
make migrate-down MIGRATE_DATABASE='mysql://user:password@tcp(localhost:3306)/go_micro?multiStatements=true&parseTime=true'
```

查看版本：

```bash
make migrate-version MIGRATE_DATABASE='mysql://user:password@tcp(localhost:3306)/go_micro?multiStatements=true&parseTime=true'
```

修复 dirty 状态：

```bash
make migrate-force VERSION=1 MIGRATE_DATABASE='mysql://user:password@tcp(localhost:3306)/go_micro?multiStatements=true&parseTime=true'
```

也可以通过环境变量传入数据库地址：

```bash
export DATABASE_URL='mysql://user:password@tcp(localhost:3306)/go_micro?multiStatements=true&parseTime=true'
make migrate-up
```

注意：应用运行时使用的 `MYSQL_DSN` 是 Go MySQL driver DSN；迁移工具使用的 `MIGRATE_DATABASE` / `DATABASE_URL` 是 `golang-migrate` 数据库 URL，两者格式不要混用。

## 4. 生产执行原则

- 迁移必须先在 staging 执行并验证。
- 生产迁移必须绑定发布单或变更记录。
- 禁止直接修改已发布的 migration 文件；新变更必须新增版本。
- 涉及大表 DDL 时需要评估锁表、执行时间和回滚方案。
- destructive migration 必须准备独立备份和人工确认。

## 5. 后续增强

- CI 中增加 migration SQL 语法校验。
- 引入临时 MySQL 容器执行 migration dry-run。
- 将迁移执行接入 staging/prod 发布流水线。
- 拆分 seed data 和 schema migration，避免生产迁移写入演示数据。
