# 缓存指标面板示例（Prometheus / Grafana）

本页提供缓存命中率的监控示例，用于定位缓存配置与策略问题。

## 一、指标说明
- `cache_hits_total{cache="order|inventory|user|..."}`  
  缓存命中次数
- `cache_misses_total{cache="order|inventory|user|..."}`  
  缓存未命中次数

标签 `cache` 来源于 key 前缀，例如：
- `order:*` -> `cache="order"`
- `inventory:*` -> `cache="inventory"`
- `user:*` -> `cache="user"`

## 二、PromQL 查询示例

1. 缓存命中率（总体）
```
sum(rate(cache_hits_total[5m])) / (sum(rate(cache_hits_total[5m])) + sum(rate(cache_misses_total[5m])))
```

2. 按模块命中率
```
sum by (cache) (rate(cache_hits_total[5m])) 
/
(
  sum by (cache) (rate(cache_hits_total[5m])) 
  + 
  sum by (cache) (rate(cache_misses_total[5m]))
)
```

3. 缓存请求总量
```
sum(rate(cache_hits_total[5m])) + sum(rate(cache_misses_total[5m]))
```

4. 缓存未命中趋势
```
sum by (cache) (rate(cache_misses_total[5m]))
```

## 三、Grafana 面板建议
- 面板 1：命中率（总体）  
  类型：Gauge/Stat
- 面板 2：命中率（按模块）  
  类型：Time series
- 面板 3：缓存请求总量  
  类型：Time series
- 面板 4：缓存未命中趋势  
  类型：Time series

## 四、优化建议
- 命中率下降时，优先排查：  
  1. TTL 是否过短  
  2. 热点数据是否被频繁失效  
  3. 空值缓存是否生效  
  4. Redis 是否稳定
