# 统一错误码说明

本文档定义了项目中使用的统一错误码体系，确保所有微服务返回一致的错误格式。

## 响应格式

```json
{
  "code": 0,
  "message": "OK",
  "data": {}
}
```

- `code`: 错误码，0表示成功
- `message`: 错误描述信息
- `data`: 响应数据

## 错误码设计原则

- **按模块分类**: 错误码按服务模块划分，每个模块占用一个区间
- **预留空间**: 每个模块预留100个错误码，方便后续扩展
- **语义明确**: 错误码和消息一一对应，便于排查问题

## 使用示例

### 成功响应
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "order_id": "ord_xxx"
  }
}
```

### 参数错误
```json
{
  "code": 40001,
  "message": "invalid request",
  "data": null
}
```

### 库存不足
```json
{
  "code": 40506,
  "message": "insufficient inventory",
  "data": null
}
```

### 订单无法取消
```json
{
  "code": 40504,
  "message": "order cannot be canceled",
  "data": null
}
```

## 代码中使用

```go
import "go-micro/pkg/errx"

// 返回错误
code, body := httpx.Fail(errx.CodeInvalidRequest, errx.MsgInvalidRequest)
c.JSON(code, body)

// 返回成功
code, body := httpx.OK(data)
c.JSON(code, body)
```
