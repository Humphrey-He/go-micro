# 项目说明文档

本文档基于当前代码实现与规划内容整理，作为交付说明与面试口径补充。

## 一、项目概述
- 业务场景：电商下单履约
- 架构模式：Go 微服务 + HTTP（对外）+ gRPC（内部）+ MQ（异步）
- 目标：完整演示订单创建主链路、库存预占、异步履约任务编排及补偿思路

## 二、服务清单与职责
- gateway-api：统一入口、鉴权、路由、Swagger 对外文档
- order-service：订单创建与状态更新、Outbox 事件
- inventory-service：库存预占/释放/确认、库存查询
- user-service：用户信息维护与查询
- task-service：消费订单事件，创建履约任务，重试与死信补偿

## 三、核心链路
1. 用户通过 gateway-api 创建订单
2. order-service 写订单 + 明细
3. order-service 调用 inventory-service 预占库存（gRPC）
4. order-service 事务写入 outbox（order.created）
5. outbox 定时发布到 MQ
6. task-service 消费消息生成履约任务

## 四、关键技术点
### 1. 内部调用（gRPC）
- 统一 IDL，强约束接口
- 更低延迟、连接复用
- 易于治理（超时、重试、拦截器）

### 2. 消息模型（RabbitMQ）
- 工作队列模型，手动 ACK
- 失败消息进入 DLQ
- 主交换机 `order.events`，死信交换机 `order.events.dlx`

### 3. Outbox 一致性
- 订单状态更新与 outbox 插入同一事务
- 由后台轮询发布，保证最终一致性

### 4. 缓存稳定性
- 空值缓存防穿透
- TTL 抖动防雪崩
- 热点变更主动删除

## 五、接口入口
- gateway-api：`/api/v1/orders`
- user-service：`/users`
- inventory-service：`/inventory/*`
- task-service：`/tasks`

## 六、运行依赖
- MySQL 8
- Redis
- RabbitMQ

## 七、配置项（核心）
- `MYSQL_DSN`：MySQL 连接串
- `REDIS_ADDR`：Redis 地址
- `MQ_URL`：RabbitMQ 连接串
- `ORDER_GRPC_ADDR` / `INVENTORY_GRPC_ADDR`

## 八、Swagger 文档

- 访问地址：`http://localhost:8080/swagger/index.html`
