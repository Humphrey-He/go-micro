# 订单管理后台 - 业务需求分析

> 本文档基于 `go-micro` 微服务电商履约系统，分析订单管理后台的业务功能需求。

## 1. 系统概述

### 1.1 背景

`go-micro` 是一个 Go 微服务电商履约系统，核心业务流程如下：

```
用户下单 → 库存预扣 → 支付 → 履约 → 完成
           ↓                      ↓
        取消/超时               退款
```

现有后端已实现完整的微服务架构，但缺少管理后台来可视化管理订单、支付、退款等业务。

### 1.2 目标

为运营/客服/管理员提供可视化的订单管理后台，支持：

- 订单查询与状态管理
- 支付状态监控与操作
- 退款处理
- 库存查看与操作
- 活动管理（优惠券/秒杀）
- 数据统计面板

---

## 2. 核心业务功能需求

### 2.1 订单管理

#### 2.1.1 订单列表

| 需求项 | 说明 |
|--------|------|
| 订单搜索 | 支持按订单号、用户ID、订单状态、时间范围搜索 |
| 订单筛选 | 按状态筛选（待支付/已支付/已完成/已取消/已退款） |
| 订单排序 | 按创建时间、金额排序 |
| 分页展示 | 每页20条，支持翻页 |
| 批量导出 | 支持CSV/Excel导出订单数据 |

**订单状态枚举：**

| 状态 | 说明 |
|------|------|
| `PENDING` | 待支付 |
| `PAID` | 已支付 |
| `FULFILLED` | 已履约完成 |
| `CANCELED` | 已取消 |
| `REFUNDED` | 已退款 |

#### 2.1.2 订单详情

| 需求项 | 说明 |
|--------|------|
| 基本信息 | 订单号、用户ID、创建时间、订单状态、总金额 |
| 商品明细 | SKU、单价、数量、小计 |
| 支付信息 | 支付单号、支付状态、支付时间 |
| 履约信息 | 任务状态、履约进度 |
| 退款信息 | 退款单号、退款状态 |
| 操作日志 | 状态变更时间线 |

#### 2.1.3 订单操作

| 操作 | 权限 | 说明 |
|------|------|------|
| 查看详情 | 全部 | 查看完整订单信息 |
| 取消订单 | 管理员/客服 | 仅允许取消未支付订单 |
| 手动履约 | 管理员 | 跳过自动履约，手动标记完成 |

---

| 查看支付详情 | 全部 | 查看支付单完整信息 |

## 3. 用户角色与权限

### 3.1 角色定义

| 角色 | 功能范围 |
|------|----------|
| `admin` | 全部功能 |
| `operator` | 订单/支付/退款查看，优惠券发放，无编辑权限 |
| `service` | 订单/支付/退款查看，订单取消 |
| `viewer` | 只读查看所有数据 |

### 3.2 权限矩阵

| 功能 | admin | operator | service | viewer |
|------|-------|----------|---------|--------|
| 订单列表查看 | ✓ | ✓ | ✓ | ✓ |
| 订单详情查看 | ✓ | ✓ | ✓ | ✓ |
| 取消订单 | ✓ | ✗ | ✓ | ✗ |
| 手动履约 | ✓ | ✗ | ✗ | ✗ |
| 退款操作 | ✓ | ✗ | ✗ | ✗ |
| 库存补货 | ✓ | ✗ | ✗ | ✗ |
| 数据导出 | ✓ | ✓ | ✗ | ✗ |

## 4. 外部系统接口

### 4.1 现有 API（通过 Gateway）

管理后台前端通过调用 Gateway REST API 获取数据：

| 接口 | 用途 |
|------|------|
| `GET /api/v1/order-views/:order_no` | 订单聚合视图 |
| `GET /api/v1/orders/:id` | 订单详情 |
| `POST /api/v1/orders` | 创建订单 |
| `POST /api/v1/payments` | 创建支付 |
| `GET /api/v1/payments/:id` | 支付详情 |
| `POST /api/v1/refund/initiate` | 发起退款 |
| `POST /api/v1/refund/status` | 退款状态 |
| `POST /api/v1/activity/coupon` | 发放优惠券 |
| `GET /api/v1/activity/status` | 活动状态 |

### 4.2 新增需求 API

| 接口 | 说明 |
|------|------|
| `GET /api/v1/admin/orders` | 后台订单列表（分页、筛选） |
| `GET /api/v1/admin/payments` | 后台支付列表 |
| `GET /api/v1/admin/refunds` | 后台退款列表 |
| `GET /api/v1/admin/inventory` | 库存列表 |
| `POST /api/v1/admin/inventory/replenish` | 库存补货 |
| `GET /api/v1/admin/stats/dashboard` | 运营看板数据 |

## 5. 非功能性需求

### 5.1 性能

- 列表查询响应时间 < 500ms
- 大数据量导出支持异步任务

### 5.2 安全

- 管理后台独立部署，与用户端分离
- JWT Token 区分管理员和普通用户
- 操作日志完整记录

### 5.3 可用性

- 支持部署多实例
- 关键操作有确认提示

---

## 6. 优先级

### Phase 1 - MVP（核心）

1. 订单列表 + 详情查看
2. 订单状态筛选 + 搜索
3. 基础运营看板

### Phase 2 - 完善

4. 支付管理 + 退款管理
5. 库存管理
6. 数据导出

### Phase 3 - 高级

7. 活动管理（优惠券/秒杀）
8. 高级统计分析
9. 自动化运营工具
